# Resource Schemes 

**0.4 update** the path to the image.toml file is also included in the resource scheme.

TeXbld accepts resource schemes in the CLI. The following schemes
are supported:

1. For a GitHub image, use `github:(owner)/(repository)/(revision)#(image file)`.
   The default revision is the latest revision on the default branch. The default image file is `image.toml`.
2. For a local image, use `local:(name)#(image file)`. The default image file is `image.toml`.
3. For a docker image, use `docker:(name)`.

# Valid Examples

## GitHub
1. `github:texbld/templates`
2. `github:texbld/templates#sample.toml`
3. `github:texbld/templates/master`
4. `github:texbld/templates/master#sample.toml`
5. `github:texbld/templates/(commit sha)`
6. `github:texbld/templates/(commit sha)#image.toml`

## Local

1. `local:myimage`
2. `local:myimage#markdown.toml`

## Docker

1. `docker:alpine`
2. `docker:pandoc/core:latest`
